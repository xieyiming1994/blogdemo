git及安装步骤

2017.01.09 Xie Yiming  热度 ℃

什么是git

[分布式的源代码版本管理工具]

源码是什么？

[在一个项目中，凡是有开发人员编写的代码都是都算是源代码。]

源代码为什么要管理起来

[让源代码可以被追溯，主要记录每次变更了什么，谁主导了这次变化。]
[协同开发，人为的维护比较麻烦]

git安装

[安装相关的Git的客户端(命令行的)]
[安装界面化的Git客户端SourceTree(tortoisegit，smartgit)]

Git简单使用

1. 使用git init初始化一个本地仓库

[在本地文件夹中添加一个.git文件夹，用于记录项目所有的变更信息。]

2. git status

[用于查看本地仓储的变更状态。第一次查看，会显示未被跟踪的文件。]
[-s参数: 输出简要的变更日志。]

3. git add .

[将没有被跟踪的文件添加到跟踪列表(git add –all)]

4. 本地忽略清单

[在.gitignore文件中添加相应的忽略清单]
[node_modules]
[/min]

5. 提交

[git commit: 将本地的变化提交到本地的仓库文件夹归档]
[git commit -m “说明”（必须有提交日志）]

6. git config

git config --global user.email "you@example.com"
git config --global user.name "Your Name"

7. git diff

[比较当前文件和文件的状态]

8. git log

[查看历史信息，查看完整历史消息]
git log -p

9. git reset –hard “哈希值”

[将代码回归]
